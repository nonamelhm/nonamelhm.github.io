# react 路由介绍

现在前端大多是 SPA（单页面应用程序）也就是只有一个 HTML 页面的应用程序。因为它的用户体验更好，所以更受欢迎。为了有效的使用单个页面来管理原来多页面的功能，前端理由应运而生

  • 前端路由功能：让用户从一个视图导航到另外一个视图
  • 前端路由是一套映射规则，在 React 中，是 URL 与组件的对应关系
  • 使用 React 路由简单来说，就是配置路径和组件（配对）

# 基本使用

  • 安装 yarn add react-router-dom/npm i react-router-dom
  • 导入路由三个核心组件、Router/Route/Link
  • 使用 Router 包含整个应用
  • 使用 Link 组件作为导航菜单（路由入口）
  • 使用 Route 组件作为配置路由规则和要展示的组件（路由规则） (V5 版本！！！)

# 常用组件说明

  • BrowserRouter：推荐用 BrowserRouter 代替 HashRouter
  • Link 组件：用于导航链接（a 标签）
  • Route 组件：指定路由展示组件的相关信息

# 路由执行过程

  • 点击 Link 组件即 a 标签，修改浏览器地址中的 url
  • React 路由监听到地址栏的 url 的变化
  • React 路由内部遍历所有 Route 组件，使用路由规则 path 与 pathname 进行匹配
  • 当路由规则 path 能够匹配到 pathname 时，就展示该 Route 组件内部内容

# 编程式导航

  • 场景：点击登录按钮，登录成功跳到后台首页
  • 编程式岛上，通过 JS 代码实现页面跳转
  • history 是 React 路由提供的，用于获取浏览器历史记录的信息
  • push (path), 跳转到某个页面，参数 path 表示要跳转的 1 路径

# 模糊匹配模式

  • 问题：当 Link 组件的 to 属性为 “/login” 时，为什么默认路由也被匹配成功
  • 默认情况下：React 路由是模糊匹配模式
  • 模糊匹配规则：只要 pathname 以 path 开头就会匹配成功 eg:/ 匹配所有 / 开头的 path /login 匹配所有 /login 开头的 path /login /login/a/b/c 等

# 精确匹配

  • 问题：默认路由任何情况都会展示，模糊匹配的文艺
  • 给 Route 组件添加 exact 属性，让其变成精确匹配
  • 精确匹配：只要 path 和 pathname 完全匹配才会展示
