# 网络协议

# 封装类协议

  1. 链路层：以太网 / PPP/HDLC/ 帧中继，常考以太网帧格式，帧中继的特点
  2. 网络层：IP 常考 IP 报头格式
  3. 传输层：TCP/UDP 常考 TCP 三次握手 拥塞和流量控制

# 以太网帧结构

  1. 目的地址 / 源地址为 MAC 地址
  2. 数据 46~1500，最小 64 字节，最长 1518 字节
  3. 校验 4 字节
  4. 先导字段不算帧长

# 帧中继

  • 第二层 本地标识 DLCL
  • 面向连接
  • CIR 承诺速率 EIR 扩展速率
  • 2~45Mbps
  • 不保证可靠的提交
  • 帧长可变 1600~4096 字节

# HDLC

  • 高级链路控制协议
  • 面向比特的数据链路层协议
  • 帧边界 “0111 1110”
  • 三种类型帧 记忆：信传 监差 无链
  • 特殊字符 REJ 传后面的 SREJ 选择接受发过的不重发

# 路由协议

# IGP（内部网关协议）

  • 用于 AS 内部

# RIP

  1. 基于 UDP520
  2. 最大 15 跳，16 跳不可达，用于小型网络
  3. 30s 周期性更新路由表，180s 更新表示不存在，240s 删除路由表
  4. RIPv1 RIPv2 区别
  5. 距离矢量协议

# OSPF

  1. 点对点网络每 10s 发送一次 hello
  2. 224.0.0.5 运行 OSPF 的路由器
  3. 224.0.0.6 OSPF 指定备用路由器 DR/BDR
  4. 链路状态路由器
  5. 基于 IP

# EGP（外部网关协议）

  • 用于 AS 之间

# BGP

  1. 基于 TCP 179
  2. 打开 open 建立邻居关系
  3. 更新 update 发送新的路由器
  4. 保持活动状态 Keepalive 对 Open 应答 / 周期性确认邻居关系
  5. 通告 Notification 报告检测到的错误

# 功能类协议

  1. ICMP：封装在 IP 报文中，传递差错控制
  2. ARP (IP-----MAC) RAPR (MAC----IP)
  3. NAT 网络地址转换，将私有地址转换为公网 IP 地址
  4. ACL 访问控制列表 实现用户访问控制
  5. STP 生成树协议，防止二层环路
  6. DHCP 动态主机配置协议，动态分配 IP 地址
  7. 其它 VRRP.DNS WWW FTP/TFTP

# 特殊地址

  1. 0.0.0.0 没有表示 0
  2. 255.255.255.255 广播地址
  3. 169.254.0.0/16 自动获取 IP 地址，DHCP 故障
  4. 127.0.0.0/8 本地环回地址
  5. 私有地址 1 个 A 类 16 个 B 类 256 个 C 类 A 类 10.0.0.0/8 10.0.0.0-10.255.255.255 B 类 172.16.0.0/12 172.16.0.0-172.31.255.255 C 类 192.168.0.0/16 192.168.0.0-192.168.255.255
  6. 224.0.0.1 所有主机
  7. 224.0.0.2 所有路由器
  8. 224.0.0.5 OSPF 路由器
  9. 224.0.0.6 DR/BDP 组播接收地址
  10. 224.0.0.9 RIPv2 组播更新地址
  11. 224.0.0.18 VRRP 组播地址
