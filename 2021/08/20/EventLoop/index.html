# 背景知识

javascript 是一个单线程语言。也就是说，同一时间只能做一件事情。

如果前一个任务非常耗时，则沟洫的任务不得不一直等待，从而导致程序假死问题。

# 同步任务和异步任务

为了防止耗时任务导致程序假死的问题，Javascript 把 待执行的任务 分成了两类。一类是 同步任务 ，一类是 异步任务 。

  • 同步任务
    • 又叫做 非耗时任务 ，指的是 主线程上 排队执行的那些任务。
    • 只有前一个任务执行完毕，才能执行下一个任务。
  • 异步任务
    • 又叫做耗时任务，异步任务由 JavaScript 委托 给 宿主环境 进行执行
    • 当异步任务执行完成时，会通知 javascript主线程 执行异步任务的 回调函数 。

执行过程

  • 同步任务由 JavaScript 执行
  • 异步任务委托给宿主环境执行
  • 已完成的异步任务对应的回调函数，会被加入到任务队列中等待执行
  • Javascript 主线程的执行栈被清空后，会读取任务队列的回调函数，次序执行
  • javascript 主线程不断重复上面第 4 步骤

# EventLoop

JavaScript 主线程 从 “任务队列“ 中读物异步任务的回调函数，放到执行栈中依次执行。这个过程是 不断循环 的，所以整个这种运行的机制又称为 EventLoop(事件循环)
