# props 深入

# 基本认识

  • 组件是封闭的，是要接首外部的数据要通过 props 来实现
  • props 作用：接收传递给组件的数据
  • 传递数据：给组件标签添加属性
  • 接收数据：函数组件通过 props 接收组件，类组件通过 this.props 接收数据
  • 注意：传非字符串数据用 {} 定义 ，可以传函数 数组 对象等

特点：

  • 可以给组件传递任意类型的数据
  • props 是只读的对象，只能读取属性的值，无法修改对象
  • 注意：使用类组件时，如果写了构造函数，应该把 props 传递 super (), 否则，无法在构造函数中获取 props!

# children 属性

  • children 属性：表示组件标签的子节点。当组件标签有子节点时，props 就会有该属性
  • children 属性与普通的 props 一样，值可以是任意值（文本、React 元素、组件甚至是函数）

# props 校验

  • 对于组件来说，props 是外来的，无法保证徐建使用者传来什么格式的数据
  • 如果传入数据合适不对，可能会导致组件内部报错
  • 关键问题：组件的使用者不知道明确的错误原因

比如下面：小明要求传入的 color 是个数组格式，而小红在使用时传了字符串，使得报错，但是不知道什么错误

props 校验

  • 允许在创建组件的时候，就指定 props 的类型、格式等
  • 作用：捕获使用组件时因为 props 导致的错误，给出明确的错误提示，增加组建的健壮性

使用步骤

  1. 安装包 prop-types (yarn add prop-types /npm i prop-types 包)
  2. 导入 prop-types 包 需要 js 文件放在第一行 import PropTypes from “prop-types”
  3. 使用组件名.propTypes={} 来给组件的 prop 添加校验规则

约束规则

  1. 常见类型：array/bool/func/number/object/string
  2. React 元素类型：element
  3. 必填项：isRequired
  4. 特定结构的对象：shape ({})

# 父传子

  • 父组件提供要传递的 state 数据
  • 给子组件标签添加属性，值为 state 中的数据
  • 子组件中通过 props 接收父组件传递数据

# 子传父

  • 父组件提供一个回调函数（用于接收数据）
  • 将该函数作为属性的值，传递给子组件

思路：利用回调函数，父组件提供回调，子组件调用，将要传递的数据作为回调函数的参数

# 兄弟之间通讯

  • 将共享状态升到最近的公共组件中，由公共父组件管理这个状态
  • 思想：状态提升
  • 功能父组件职责：1 提供共享状态 2 提供操作共享状态的方法
  • 要通讯的子组件只需通过 props 接收状态或操作状态的方法

# 组件生命周期

  • 意义：组件的生命周期有助于理解组建的运行方式、完成更复杂的组件功能、分析组件的错误原因等
  • 组件的生命周期：组件从被创建到挂载到页面中运行，再到组件不用时卸载的过程
  • 生命周期的每个阶段总是伴随着一些方法调用，这些方法就是生命周期的钩子函数
  • 钩子函数的作用：为开发人员在不同阶段操作组件提供了时机

生命周期三个阶段

  1. 每个阶段的执行时机
  2. 每个阶段钩子函数的执行顺序
  3. 每个阶段钩子函数的作用

# HOC 高阶组件

# 理解

  • 目的：实现状态逻辑复用
  • 采用包装装饰模式，比如说，手机壳
  • 手机：获取保护功能
  • 手机壳：提供保护功能
  • 高阶组件就相当于手机壳，通过包装组件，增强组件功能

实现

  • 高阶组件（HOC,Higher-Order-Component) 是一个函数，接收要包装的函数，返回增强后的组件
  • 高阶函数内部创建一个类组件，在这个类组件中提供复用的状态逻辑代码，通过 props 将复用的状态传递给被包装组件 WrappedComponent

# 使用

  • 创建一个函数，名字约定以 With 开头
  • 指定函数参数（作为要渲染的组件，应大写字母开头）
  • 在函数内部创建一个类徐建，提供复用的状态逻辑代码，并返回
  • 在该组件中渲染参数组件，同时将状态通过 props 传递给参数组件

# setState()

  • setState 是异步更新数据的
  • 注意：使用该数据时，其后面的 setSate 不依赖于前面的 setState ()
  • 多次调用 setState () 只会触发一次重新渲染

推荐语法

  • 使用 setState (state,props)=>{}) 语法
  • 参数 state: 表示 最新的state
  • 参数 props: 表示 最新的props

第二个参数

  • 场景：在状态更新后立即执行某个操作
  • 语法：setState (update [,callback])

# 组件性能优化

1. 减轻 state: 只存储组件渲染相爱难相关的数据

  • 注意：不用做渲染的数据 不要放在 state 中
  • 对于这种需要在多个方法中渲染的，直接放 this

2. 组件更新机制：父组件更新会引起子组件也被更新

  • 使用钩子函数 shouldComponenentUpdate (nextProps,nextState)

  • nextState 更新后的数据 this.state 更新前数据

  • nextProps 更新后的数据 this.props 更新前数据

  • 触发时机：更新阶段的钩子函数，组件更新渲染时执行（shouldComponentUpdate------>render)

    # 纯组件

    • 不需要手动比较 shouldComponentUpdate
    • 纯组件内部通过对比前后两次 props 和 state 的值，来决定是否重新渲染组件

  • 说明：纯组件内部的对比是 shallow compare (浅层对比)

  • 对于值类型来说，比较两个值是否相同（直接赋值即可）

  • 对于引用类型来说，只比较对象的引用地址是否相同

  • 对象：不要直接修改数据，而是通过创建新对象的方法
  • 数组：不要用数据的 push/unshilft 直接修改当前数组，而是用 cincat slice 等返回新数据的方法。总结：创建新的不要直接修改

# 虚拟 DOM 和 Diff 算法

  • React 更新视图的思想是：只要 state 变化就重新渲染视图
  • 问题：组件中只有一个 DOM 元素需要更新时，也得把组件的内容重新渲染到页面中
  • 理想状态：部分更新，只更新有变化的地方
  • 问题：React 是如何做到部分更新的？虚拟 DOM 和配合 Diff 算法

# 虚拟 DOM

  • 本质上就是一个 JS 对象，用来描述你希望在屏幕上看到的内容
  • 组件 render 后，根据状态和 JSX 结构生成虚拟 DOM 对象

# Diff 算法

  • 根据虚拟 DOM，找不同，渲染变化的地方

# 执行过程

  1. 执行渲染时，React 会根据 state, 创建一个虚拟 DOM 树
  2. 根据虚拟 DOM 生成真正的 DOM，渲染到页面中
  3. 当数据变化后（调用 setState ()），重新根据新的数据，创建出新的虚拟 DOM 树
  4. 与上一次得到的虚拟 DOM 树进行对比，使用 Diff 算法找出不同，得到需要更新的内容
  5. 最终，React 只将变化的内容更新（patch）到 DOM 中，重新渲染到页面中
