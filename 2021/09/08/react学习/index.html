# React 概述

  • React 是一个构建用户界面的 JavaScript 库

  • 用户界面：HTML 页面（前端）

  • React 主要用来写 HTML 页面，构建 Web 页面

如果从 MVC 角度来说，React 仅仅是视图层（V)，也就是只负责视图的渲染，而并非提供了完整的 M 和 C 的功能。

# React 的特点

  • 声明式
  • 基于组件
  • 学习一次，随处使用 web 移动端 VR 技术

# JSX

# 简介

JSX 是 JavaScript XML 的简写

表示在 JavaScript 代码中写 XML（HTML）格式的代码。

优势：

  • 声明式语法更加直观

  • 与 HTML 结构相同

  • 降低了学习成本，提升开发效率。

因为 React.createElement () 很繁琐 很难看出结构，则出现了 JSX 替代

# 使用

使用 JSX 替代 React.createElement ()

# 原因

为什么 JSX 能在脚手架使用 JSX 语法？

  1. JSX 不是标准的 ECMAScript 语法，它是 ECMAScript 语法扩展
  2. 需要使用 babel 编译处理后，才能在浏览器环境中使用
  3. create-react-app 脚手架已经默认有该配置，无需手动配置
  4. 编译 JSX 语法的包为：@babel/preset-react

# 注意点

  1. React 元素的属性名使用驼峰命名法
  2. 特殊属性名：class->className 、for->htmlFor、tabindex->tabIndex
  3. 没有子节点的 React 元素可以用 /> 结束
  4. 推荐：使用小括号包裹 JSX，从而避免 JS 中的自动插入分号陷阱

# 总结

  1. JSX 是 React 的核心内容
  2. JSX 表示在 JS 代码中写 HTML 结构，是 React 声明式的体现
  3. 使用 JSX 配合嵌入的 JS 表达式，条件渲染。列表渲染，可以描述任意 UI 结构
  4. 推荐使用 className 的方式给 JSX 添加样式
  5. React 完全利用 JS 语言自身能力表写 UI，而不是造轮子用指令的各种方法增强 HTML 功能。

# React 组件

# 介绍

  • 组件是 React 的一等公民，使用 React 就是在使用组件
  • 组件表示页面的部分功能
  • 组件多个组件实现完整的页面功能
  • 特点：可复用、独立、可组合

# 函数组件

使用函数创建组件

  • 函数组件：使用 js 的函数（或者箭头函数）创建组件
  • 约定 1：函数名称用大写字母开头
  • 约定 2：函数约定必须有返回值，表示该组件的结构
  • 如果返回值为 null，表示不渲染任何内容

# 类组件

使用类创建组件

  • 函数组件：使用 js 的函数（或者箭头函数）创建组件
  • 约定 1：类名称用大写字母开头
  • 约定 2：类名称必须继承 React.Component 父类，从而可以使用父类提供的方法或者属性
  • 约定 3：类组件比如提供 **render ()** 方法
  • 约定 4：render () 必须有返回值，表示该组件的结构
  • 如果 render () 返回值为 null，表示不渲染任何内容

# 组件抽离

  • 组件作为一个独立的各个体，一般都会放到一个单独的 JS 文件中

# 事件对象

  • 可通过事件处理程序的参数获取到事件对象
  • React 中的事件对象叫做：合成事件（对象）
  • 合成事件：兼容所有浏览器，无需担心跨浏览器的兼容问题

# 有状态组件和无状态组件

  • 函数组件又叫做无状态组件，类组件又叫做有状态组件

  • 状态即数据

  • 函数组件没有自己的状态，只负责数据展示 (静)

  • 类组件有自己的状态，负责更新 UI，让页面 UI 动起来

  • 需要跟用户交互必须用有状态组件，即类组件

  • 不需要的跟用户交互与数据打交道用无状态组件，即函数组件

# state 和 setState

  • 状态 state 即数据，是组件内部的私有数据，只能在组件内部使用
  • state 的值是对象，表示一个组件中可以有多个数据

# state

  • 状态即数据
  • 状态是私有的，只能在组件内部使用
  • 通过 this.state 来获取状态

# setState

  • 状态是可变的
  • 语法：this.setState ({修改的数据})
  • setState () 作用：1 修改 state 2 更新 UI
  • 注意：不要直接修改 state 中的值，这是错误的
  • 思想：数据驱动视图

# 表单处理

  • HTML 中表单元素是可输入的 input, 也就是有自己的可变状态
  • React 中可变状态通常保存在 state 中，并且只能通过 setState () 方法来修改
  • React 将 state 与表单元素的 value 值绑定在一起，由 state 的值来控制表单元素的值
  • 受控组件：其值收到 React 控制的表单元素

受控组件

  • 在 state 中添加一个状态，作为表单元素的 value 值
  • 给表单元素绑定 change 事件，将表单元素的值设置为 state 的值
